ความเสี่ยงด้านความปลอดภัย (Security Risk)

🏥 AYODIA
ความเสี่ยงแนวทางการจัดการความเสี่ยง
ข้อมูลผู้ป่วยและข้อมูลด้านสุขภาพรั่วไหล หรือถูกเข้าถึงโดยไม่ได้รับอนุญาต
  • ควบคุมสิทธิ์การเข้าถึงข้อมูลตามบทบาทผู้ใช้ (Role-based Access Control)
  • ดำเนินการตาม พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล (PDPA) ในฐานะผู้ประมวลผลข้อมูล
ข้อมูลระหว่างจัดเก็บและรับส่งไม่ได้เข้ารหัส เสี่ยงต่อการถูกดักจับ
  • เข้ารหัสข้อมูล (Encryption) ทั้งขณะจัดเก็บและขณะรับส่ง
ระบบมีช่องโหว่ หรือถูกโจมตีทางไซเบอร์ (Ransomware / SQL Injection)
  • ทำ Vulnerability Assessment / Penetration Test ก่อน Go-live
  • อัปเดตแพตช์และเฝ้าระวังความมั่นคงปลอดภัยอย่างต่อเนื่อง
ไม่สามารถสืบย้อนการเข้าถึงหรือแก้ไขข้อมูลได้ เมื่อเกิดเหตุ
  • จัดเก็บบันทึกการใช้งานระบบ (Audit Log) ครบถ้วนและตรวจสอบย้อนหลังได้
ข้อมูลรั่วไหลระหว่างการโอนย้ายข้อมูล (Data Migration)
  • โอนย้ายข้อมูลแบบเข้ารหัส และลบไฟล์ข้อมูลชั่วคราวหลังใช้งาน
  • มีขั้นตอนแจ้งเหตุละเมิดข้อมูลส่วนบุคคล (Data Breach Notification)

ความเสี่ยงด้านคุณภาพ (Quality Risk)

🏥 AYODIA
ความเสี่ยงแนวทางการจัดการความเสี่ยง
ข้อบกพร่องระดับร้ายแรง (Critical/High) หลุดขึ้นใช้งานจริง
  • กำหนดเกณฑ์ตรวจรับ: ต้องไม่มีข้อบกพร่อง Critical/High ค้างในกระบวนการหลัก
  • ทำ Regression Test ทุกครั้งก่อน Deploy ขึ้นระบบ
การทดสอบไม่ครอบคลุม โดยเฉพาะโมดูลเฉพาะทาง (Smart IPD / เฉพาะทาง)
  • จัดทำ Requirement Traceability Matrix โยงข้อกำหนด → Test Case ให้ครบทุกข้อ
ข้อมูลหลังการโอนย้ายไม่ถูกต้องหรือไม่ครบถ้วน กระทบการรักษา
  • ทำ Data Reconciliation 100% สำหรับข้อมูลวิกฤต (ประวัติแพ้ยา / ยา active)
  • ซ้อมโอนย้าย (Mock Migration) ก่อน Go-live เพื่อยืนยันความถูกต้อง
ประสิทธิภาพระบบช้าเมื่อใช้งานจริง (ผู้ใช้พร้อมกันจำนวนมาก / ข้อมูลปริมาณมาก)
  • ทำ Performance / Load Test ตามเกณฑ์เวลาตอบสนองที่กำหนดในแผนการทดสอบ
ผลการทดสอบยอมรับ (UAT) ไม่สะท้อนการใช้งานจริง
  • UAT ต้องลงนามรับรองผลโดย Business Process Owner รายโมดูล
  • ใช้ชุดข้อมูลและสถานการณ์ทดสอบที่ใกล้เคียงการใช้งานจริง
