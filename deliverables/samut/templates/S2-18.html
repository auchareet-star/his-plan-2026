2ขั้นตอนการติดตั้ง (Deployment Steps — Repeatable)

2.1 ระบบ Web + Backend (HA.OS)

  • ดึง source/artifact เวอร์ชัน release ที่อนุมัติ (tag: __)
  • ตั้งค่าตัวแปรสภาพแวดล้อม/ไฟล์ config ตามข้อ 3
  • Build / Migrate schema ฐานข้อมูล (รัน DB script ตามลำดับ)
  • Deploy ขึ้น App Server + restart service
  • ตรวจ health check endpoint ตอบ 200 OK
$ git checkout release-vX.Y.Z $ cp .env.production .env # ตั้งค่าตามข้อ 3 $ ./scripts/db-migrate.sh # รัน migration ตามลำดับ $ ./scripts/build.sh # build artifact $ ./scripts/deploy.sh prod # deploy + restart service $ curl -fsS https://his.________/healthz # คาดหวัง 200 OK

4การตรวจสอบหลัง Deploy (Smoke Test)

  • เปิดหน้าเข้าสู่ระบบ Web ได้ + login ผ่าน
  • ลงทะเบียนผู้ป่วยทดสอบ + ค้นหา patient bank ได้
  • สร้างนัดหมาย/คิว (Queue) ได้
  • Admit ผู้ป่วย IPD + บันทึก eMAR (Smart IPD) ได้
  • สั่ง/จ่ายยา + ตัดสต็อกคลังยา + ออกใบเสร็จ (งานเรียกเก็บ) ได้
  • เรียกดูภาพรังสีจาก PACS + ส่ง/รับผล LIS-RIS ได้
  • ตรวจสอบสิทธิ์/ส่งเบิก สปสช. ได้
  • ไม่มี error ระดับ Critical/High ใน log

5ขั้นตอนการถอยกลับ (Rollback)

  • หยุด service เวอร์ชันใหม่
  • กู้คืน artifact/เวอร์ชันก่อนหน้า (tag: __)
  • ย้อน DB migration / restore backup (ไฟล์: __)
  • ตั้ง config กลับเวอร์ชันเดิม + restart
  • ตรวจ smoke test เวอร์ชันเดิมผ่าน + แจ้งทีม
