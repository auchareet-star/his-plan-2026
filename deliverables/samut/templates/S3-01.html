3ทีม Onsite และความรับผิดชอบ (Rollout Team)

ทีมเดินสาย rollout ต่อจุด — กำหนดผู้นำต่อจุดและ backup · ตารางนี้สรุปบทบาทหลัก

บทบาทผู้รับผิดชอบหน้าที่ระหว่าง Rollout
PM / กำกับ Rolloutอาร์ต (PM เมืองสมุทร)คุมตารางเปิดใช้ทุกจุด, ตัดสิน Go/No-Go, ประสานเจ้าของงาน
Migration LeadSenior Imp เตทีม (HosXP)นำการโอนย้ายข้อมูล HosXP→HA.OS ต่อจุด + reconciliation
Dev Lead / แก้ปัญหา onsiteซี (Dev Lead)แก้ defect ด่วน, ปรับ config, สนับสนุน hotfix
Implementer / สอนใช้งานหน้างาน__ติดตั้ง, smoke test, ช่วยผู้ใช้วันเปิดใช้ (hypercare)
ผู้แทนเจ้าของงานต่อจุด__ยืนยันผู้ใช้พร้อม, ลงนามรับมอบจุดบริการ

4เกณฑ์เปิดใช้รายจุด (Go/No-Go Criteria per Site)

ทุกข้อต้องผ่าน (Go) ก่อนเปิดใช้แต่ละจุด · หากข้อใดไม่ผ่าน = No-Go เลื่อนเปิดและบันทึกเหตุผล

  • Smoke test รายจุด (ข้อ 7) ผ่านครบทุกข้อ
  • ผู้ใช้หน้างานพร้อม (อบรมแล้ว, รับสิทธิ์เข้าใช้, มีผู้ประสานต่อจุด)
  • Network / อุปกรณ์ปลายทางพร้อม (เชื่อมต่อ App Server เสถียร, จอแสดงคิวพร้อม)
  • ข้อมูล Migration HosXP→HA.OS ของจุดนั้น reconciliation ครบถ้วน
  • ไม่มี defect ระดับ Critical/High ค้างที่กระทบจุดบริการนั้น
  • มีแผน rollback (ข้อ 8) และทีม onsite/hypercare พร้อมในวันเปิดใช้

5ขั้นตอนการเปิดใช้ต่อจุด (Repeatable per Site)

ระบบ HA.OS เป็น Web-based HIS ส่วนกลาง — ต่อแต่ละจุดเน้นการเตรียมข้อมูล/สิทธิ์/network และตรวจรับหน้างาน ไม่ใช่การติดตั้งแอปแยกเครื่อง

  • ยืนยันเวอร์ชัน HA.OS บน Production ที่อนุมัติ (release tag: __)
  • เตรียม Master Data เฉพาะจุด (ผังบริการ/ห้องตรวจ/คิว) ตามข้อ 6
  • โอนย้าย/ตรวจสอบข้อมูลผู้ป่วยของจุดนั้นจาก HosXP + reconciliation
  • สร้างบัญชีผู้ใช้/สิทธิ์ตามบทบาทหน้างาน + ทดสอบ login
  • ตรวจ network/เครื่องลูกข่าย/จอแสดงคิว เชื่อม App Server ได้
  • รัน smoke test รายจุด (ข้อ 7) → ตัดสิน Go/No-Go (ข้อ 4)
  • เปิดใช้จริง + hypercare หน้างาน + บันทึกผลในข้อ 9
$ # ตรวจระบบส่วนกลาง HA.OS พร้อมก่อนเปิดจุดใหม่ $ curl -fsS https://his.msm.local/healthz # คาดหวัง 200 OK $ curl -fsS https://his.msm.local/api/sites/<site_id>/status # จุดบริการ active $ ./scripts/site-enable.sh <site_id> # เปิดสิทธิ์จุดบริการ $ ./scripts/migrate-verify.sh <site_id> # reconciliation ข้อมูลจุดนั้น $ curl -fsS https://his.msm.local/api/integration/nhso/ping # ทดสอบเชื่อม สปสช.

7การตรวจสอบหลังเปิดใช้รายจุด (Smoke Test per Site)

  • เปิดหน้าเข้าสู่ระบบ HA.OS ได้ + login ผู้ใช้ของจุดนั้นผ่าน
  • ลงทะเบียน/ค้นหาผู้ป่วย + ตรวจสิทธิ์ สปสช. ของจุดนั้นได้
  • สร้าง Visit / คิว (Patient Self Visit & Queue) + จอแสดงคิวทำงาน
  • สั่ง/จ่ายยา (คลังยาและวัสดุ) + ออกใบเสร็จ/งานเรียกเก็บได้
  • ส่ง/รับผล LIS + เรียกดูภาพ PACS/RIS ได้
  • รับ-ส่งข้อความ HL7 + เชื่อม สปสช. ทำงานถูกต้อง
  • ข้อมูลที่ migrate จาก HosXP ของจุดนั้นแสดงถูกต้อง (สุ่มตรวจ)
  • ไม่มี error ระดับ Critical/High ใน log

8ขั้นตอนการถอยกลับรายจุด (Rollback)

  • ปิดสิทธิ์/หยุดเปิดใช้จุดบริการนั้น (disable site) + แจ้งผู้ใช้หน้างาน
  • สลับจุดนั้นกลับใช้ระบบเดิม HosXP ชั่วคราว (กระดาษ/แผนสำรอง)
  • ย้อนข้อมูล/ตั้งค่าของจุดนั้นกลับ (restore backup ไฟล์: __)
  • วิเคราะห์สาเหตุ + แก้ defect ก่อนตั้งวันเปิดใช้ใหม่
  • ตรวจ smoke test รายจุดผ่านอีกครั้ง + แจ้งทีม

เวลา Rollback เป้าหมาย (RTO): __ นาที — การ rollback จุดเดียวไม่กระทบจุดอื่นที่เปิดใช้แล้ว (ระบบส่วนกลางยังทำงาน)
