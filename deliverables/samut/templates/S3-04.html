3กรรมสิทธิ์, License และสิทธิ์ (Ownership, License & Rights)

⚠️ ข้อกำหนดทรัพย์สินทางปัญญา (สำคัญที่สุด): กรรมสิทธิ์ใน Source Code ของระบบ HIS (HA.OS) เป็นของผู้รับจ้าง — บริษัท อโยเดีย จำกัด และ ไม่ โอนให้แก่โรงพยาบาล โรงพยาบาลได้รับเพียง สิทธิ์การใช้งาน (License to Use) ระบบเท่านั้น · ส่วน ข้อมูลผู้ป่วย (Patient Data) เป็นกรรมสิทธิ์ของโรงพยาบาล แยกออกจากกรรมสิทธิ์ Source Code โดยชัดเจน
  • 3.1 กรรมสิทธิ์ Source Code: Source Code ทั้งหมดยังคงเป็นกรรมสิทธิ์ของผู้รับจ้าง (บริษัท อโยเดีย จำกัด) — ไม่มีการโอนกรรมสิทธิ์ให้โรงพยาบาล โรงพยาบาลได้รับสิทธิ์การใช้งานระบบ (License to Use) ภายใต้เงื่อนไขสัญญา และเข้าถึง Source Code ได้เฉพาะเมื่อเข้าเงื่อนไขการปล่อย Escrow (ดูข้อ 4) __________
  • 3.2 สิทธิ์การใช้งานของโรงพยาบาล (License Scope): ระบุขอบเขตสิทธิ์ใช้งาน — จำนวนจุดบริการ (9 จุด), ระยะเวลา, การใช้งานภายในองค์กร, ข้อห้ามทำซ้ำ/แจกจ่าย/ดัดแปลงนอกขอบเขต __________
  • 3.3 กรรมสิทธิ์ข้อมูลผู้ป่วย: ข้อมูลผู้ป่วยและข้อมูลที่เกิดจากการให้บริการทั้งหมด เป็นกรรมสิทธิ์ของโรงพยาบาล ผู้รับจ้างไม่มีสิทธิ์ในข้อมูลดังกล่าวนอกเหนือจากที่จำเป็นต่อการให้บริการ/MA

4การวาง Source Code ใน Escrow แบบเข้ารหัส (Encrypted Source Code Escrow)

⚠️ หลักการ Escrow: เนื่องจากกรรมสิทธิ์ Source Code เป็นของผู้รับจ้าง การคุ้มครองความต่อเนื่องของโรงพยาบาลทำผ่านการวาง Source Code แบบเข้ารหัส (Encrypted Escrow) ไว้กับ ผู้รับฝากที่เป็นกลาง (Escrow Agent) โดย archive ที่เข้ารหัสและ กุญแจถอดรหัส (decryption key) เก็บแยกกัน · Source Code จะถูก ปล่อย (release) ให้โรงพยาบาลใช้งานได้เฉพาะเมื่อเข้า เงื่อนไขการปล่อย (Release Triggers) ที่กำหนดเท่านั้น เช่น ผู้รับจ้างเลิกกิจการ หรือไม่ปฏิบัติตามภาระผูกพัน MA

เงื่อนไขปล่อย (Release Triggers)__ — เช่น ผู้รับจ้างเลิกกิจการ / ล้มละลาย, ไม่ปฏิบัติตามภาระผูกพัน MA หรือการรับประกัน, หยุดให้บริการเกินกำหนด
กระบวนการปล่อย (Release Process)__ — วิธียืนยันเงื่อนไข, ผู้มีอำนาจอนุมัติ, การมอบกุญแจถอดรหัสให้โรงพยาบาล
รอบปรับปรุง Escrow (Update Cycle)__ — เช่น ทุกครั้งที่ขึ้นเวอร์ชันใหญ่ / รายไตรมาส ตลอดอายุ MA
ค่าใช้จ่าย / ผู้รับผิดชอบ__
  • ตรวจ archive เข้ารหัสและ checksum ตรงกับ Source Code เวอร์ชันที่ส่งมอบ
  • ยืนยันว่า decryption key เก็บแยกจาก archive และอยู่ในความดูแลตามที่ตกลง
  • ระบุเงื่อนไขการปล่อย (Release Triggers) ครบถ้วนและชัดเจนในสัญญา Escrow

5คำแนะนำ Build / Deploy (สำหรับใช้เมื่อปล่อย Escrow)

สรุปขั้นตอนสำหรับให้ทีม รพ. สร้างและติดตั้งระบบเองได้ เมื่อเข้าเงื่อนไขปล่อย Escrow แล้วเท่านั้น (รายละเอียดเต็มอยู่ในชุดคู่มือ Build/Deploy)

# Web / API (HA.OS) $ git clone <repo-url> && cd haos-api $ cp .env.example .env # กรอกค่าตามตาราง config $ <install deps> # ตามไฟล์ lock $ <run db migrate> $ <build> && <deploy> # Integration (สปสช. / HL7) $ cd haos-integration $ <config endpoint สปสช./HL7> $ <build> && <deploy> # Data Migration (HosXP → HA.OS) $ cd migration-hosxp $ <run migration scripts> # reconciliation 100% ก่อนใช้งานจริง
  • ระบุ prerequisite (เวอร์ชัน toolchain/SDK) ครบ
  • ทดสอบ build จากเครื่องสะอาดสำเร็จก่อนวาง Escrow
